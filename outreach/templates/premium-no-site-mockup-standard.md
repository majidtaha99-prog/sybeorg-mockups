# Premium no-site / weak-site mockup standard

Use this for prospects where SYBEORG is showing a first website, replacement for a placeholder, or obvious upgrade over a weak current site.

## Required sections

1. **Branded loader**
   - Full-screen opening loader before the site appears.
   - Use the company logo if publicly available and usable.
   - If no logo is available, use a restrained temporary monogram that can be swapped later.

2. **Premium hero**
   - Large confident headline tied to the prospect's current problem.
   - One primary tap-to-call CTA and one secondary contact/estimate CTA.
   - Use a premium palette pulled from the business when available; otherwise create a restrained industry-appropriate palette.

3. **Interactive customer path**
   - Include a moving/interactive element such as service routing, issue selector, estimate path, ticker, or animated ambient detail.
   - Motion should feel expensive and subtle, not gimmicky.

4. **Local map section**
   - Include an embedded map or styled map treatment.
   - Add service-area/location pills and an open-map link.
   - Verify coordinates before sending outreach.

5. **Contact us section**
   - Include name, phone, and message fields.
   - Repeat the call CTA.
   - Make the production routing clear internally; do not imply a live form works if it does not.

6. **Service lanes**
   - Clean service cards/rows with plain-language labels.
   - Do not invent services, specialties, awards, reviews, ratings, warranties, licenses, or years in business.

7. **Credibility guardrails**
   - State internally that only approved/verified facts should be used.
   - Public copy should not sound defensive or mention AI.

## Quality bar

- Must look premium on mobile first.
- Must have no horizontal overflow at 390px width.
- Must pass `python scripts/qa_static.py`.
- Must not use external JavaScript.
- Must avoid generic AI copy like "transform your digital presence" or "unlock your potential."
- Must not be sent without Stark approval.
